# 断点续训功能修改记录

**日期**: 2026-02-27

## 概述

为 Search-R1 的 PPO/GRPO 训练流程添加了断点续训（resume from checkpoint）功能，支持训练中断后从最近的 checkpoint 恢复训练，并在同一个 wandb 曲线上继续记录指标。

---

## 修改的文件

### 1. `verl/trainer/ppo/ray_trainer.py`（核心修改）

#### 1.1 `__init__` 末尾

新增 resume 初始化逻辑：
- 调用 `_peek_resume_wandb_run_id()` 在 logger 初始化前获取 wandb run_id
- 调用 `_setup_resume_from_checkpoint()` 设置模型路径和训练状态

#### 1.2 新增方法 `_peek_resume_wandb_run_id()`

在 `_init_logger()` 之前读取 `trainer_state.json` 中的 `wandb_run_id`，传给 wandb 以实现同一曲线续接。

#### 1.3 新增方法 `_setup_resume_from_checkpoint()`

功能：
- 自动检测 `resume_from_checkpoint` 目录下最新的 `actor/global_step_XXX/` checkpoint
- 用 `open_dict` 更新 config 中 actor 和 critic 的 `model.path` 为 checkpoint 路径
- 读取 `trainer_state.json` 恢复 `global_steps`、`kl_coef`、`actor_kl_coef`、`actor_lr_scale`

#### 1.4 修改 `_init_logger()`

传入 `resume_run_id` 参数给 `Tracking`，使 wandb 能 resume 到同一 run。

#### 1.5 修改 `_save_checkpoint()`

在保存 actor/critic 模型权重后，额外保存 `trainer_state.json` 到 actor checkpoint 目录：
```json
{
  "global_steps": 100,
  "kl_coef": 0.001,
  "actor_kl_coef": 0.05,
  "actor_lr_scale": 1.0,
  "wandb_run_id": "c310urtw"
}
```

#### 1.6 修改 `fit()`

- 根据 `resume_step` 计算 `start_epoch` 和 `steps_to_skip`
- 跳过已完成的 epoch 和 batch（`continue` 跳步）
- resume 时跳过 `val_before_train`
- `global_steps` 从 `resume_step + 1` 开始

---

### 2. `verl/utils/tracking.py`

`Tracking.__init__()` 新增 `resume_run_id` 参数：
- 若 `resume_run_id` 非空，调用 `wandb.init(id=resume_run_id, resume='must')` → 在原 wandb run 上继续
- 否则正常创建新 run

---

### 3. `verl/trainer/config/ppo_trainer.yaml`

`trainer:` 下新增：
```yaml
resume_from_checkpoint: null
```

---

### 4. 训练脚本

| 新脚本 | 基于 | 区别 |
|---|---|---|
| `train_ppo_resume.sh` | `train_ppo.sh` | `RESUME_FROM_CHECKPOINT=verl_checkpoints/$EXPERIMENT_NAME`，日志文件名加 `_resume` |
| `train_grpo_resume.sh` | `train_grpo.sh` | 同上 |

原脚本 `train_ppo.sh` 和 `scripts/nq_hotpotqa/v0.3/train_ppo_format.sh` 也已添加 `RESUME_FROM_CHECKPOINT=null` 变量和 `trainer.resume_from_checkpoint` 参数（默认不开启）。

## 使用方法

### 从头训练（默认行为不变）

```bash
bash train_ppo.sh   # RESUME_FROM_CHECKPOINT=null，正常训练
```

### 断点续训

```bash
bash train_ppo_resume.sh   # 自动检测 verl_checkpoints/$EXPERIMENT_NAME 下最新 checkpoint
```

或手动指定 checkpoint 目录：
```bash
export RESUME_FROM_CHECKPOINT=verl_checkpoints/my-experiment
bash train_ppo.sh
```

### 机制

```
verl_checkpoints/my-experiment/
├── actor/
│   ├── global_step_100/
│   │   ├── model.safetensors      ← actor 模型权重
│   │   ├── config.json
│   │   └── trainer_state.json     ← 训练状态（step, kl_coef, wandb_run_id）
│   └── global_step_200/
│       └── ...
└── critic/
    ├── global_step_100/           ← critic 模型权重
    └── global_step_200/
```

---

## wandb 行为

| 场景 | 行为 |
|---|---|
| 首次训练 | 创建新 wandb run，记录 step 1 → N |
| 断点续训 | 通过保存的 `wandb_run_id` resume 到同一 run，曲线从断点继续 |
| checkpoint 之后的步数 | 如训练到 step 150 但 checkpoint 在 step 100，续训从 101 开始，101-150 的旧数据被新值覆盖 |

建议将 `save_freq` 设为合理值（如 50）以减少断点续训时丢失的步数。

---

## 已知限制

1. **Optimizer 状态不恢复**：FSDP 全量保存只存模型权重，Adam 动量/方差从零重置。对 RL 训练影响很小。
2. **DataLoader 顺序**：由于 `shuffle=True`，resume 后同一 epoch 内的 batch 顺序可能不同（但不影响训练收敛性）。
3. **需要首次训练时保存过 checkpoint**：如果 `save_freq=-1` 或训练在第一次保存前就中断，则无法 resume。
